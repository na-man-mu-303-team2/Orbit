# 참여 장표 개발 완료 검수 시나리오

## 1. 목적

이 문서는 에디터의 `사전 질문`, `실시간 투표`, `만족도 조사`가 아래 전체 흐름에서 정상 동작하는지 사람이 직접 확인하기 위한 인수 테스트 문서다.

```text
장표 추가 → 문항 편집 → 캔버스 반영 → 응답 열기 → 청중 제출
→ 발표자 집계·검수 → 응답 마감 → 결과 공개 → 결과 장표 확인
```

체크 결과는 항목마다 다음 중 하나로 기록한다.

- `[x]` 통과
- `[ ]` 미검수
- `[FAIL]` 기대 결과와 다름
- `[BLOCKED]` 현재 UI 또는 기능이 없어 검수할 수 없음

모든 핵심 항목이 `[x]`이고 `BLOCKED` 항목을 제품 범위에서 제외하거나 구현해야 해당 장표를 개발 완료로 판정한다.

### 1.1 가장 빠른 전체 검수 순서

세 장표를 한 번에 검수할 때는 아래 순서를 권장한다. 같은 발표 세션을 재사용하되 청중 브라우저 A, B, C는 서로 다른 프로필 또는 시크릿 창으로 유지한다.

1. 에디터에서 `사전 질문`, `실시간 투표`, `만족도 조사`를 차례로 추가한다.
2. 이 문서의 예시 입력값으로 각 장표를 구성하고 캔버스·썸네일 반영을 확인한다.
3. 발표 세션을 한 번 만들고 각 장표의 직접 링크 또는 QR을 준비한다.
4. `사전 질문`을 열어 A, B가 응답한 뒤 검수·마감·공개를 확인한다.
5. `실시간 투표`를 열어 A, B, C가 응답한 뒤 집계·마감·공개를 확인한다.
6. `만족도 조사`를 열어 A, B, C가 응답한 뒤 평균·분포·주관식 검수·공개를 확인한다.
7. 각 장표에서 `결과 숨기기`와 `응답 다시 열기`를 확인한다.
8. 새로고침 후 장표 정의, 운영 상태, 응답 수가 유지되는지 확인한다.
9. 세 장표 각각 `새 실행 버전 만들기`를 실행해 이전 결과는 보존되고 새 버전은 응답 수 `0`으로 시작하는지 확인한다.
10. 6장의 화면 증거를 남긴다.
    - 발표자: 사전 질문 검수, 투표 집계, 만족도 집계
    - 청중: 사전 질문 접수 완료, 투표 공개 결과, 만족도 접수 완료

전체 검수 예상 소요 시간은 약 25~35분이다.

## 2. 사전 준비

### 2.1 실행 환경

- Web: `http://localhost:5173`
- API: `http://localhost:3000`
- 테스트할 프로젝트와 덱 1개
- 발표자용 브라우저 1개
- 청중용 시크릿 창 또는 다른 브라우저 3개
  - 동일 브라우저 프로필은 같은 청중으로 인식될 수 있으므로 투표 집계 검수에는 서로 다른 프로필을 사용한다.

### 2.2 공통 준비 순서

1. 발표자 브라우저에서 프로젝트 에디터를 연다.
2. 왼쪽 슬라이드 패널 아래 `+`를 누른다.
3. 검수할 장표 유형을 선택한다.
4. 오른쪽 `속성` 패널에서 제목, 설명, 문항을 입력한다.
5. 입력값이 중앙 캔버스와 왼쪽 썸네일에 즉시 반영되는지 확인한다.
6. 오른쪽 패널의 `참여 장표 운영` 영역까지 내려간다.
7. `발표 세션이 없습니다.`가 보이면 `발표 세션 만들기`를 누른다.
8. 장표별 운영 상태가 `준비`, 응답 수가 `0`인지 확인한다.
9. 필요하면 왼쪽 `+` 메뉴에서 `참여 결과 연결` 장표를 추가하고 원본 참여 장표를 연결한다.

### 2.3 공통 상태 전이

| 순서 | 발표자 동작 | 기대 상태 | 청중 화면 |
| --- | --- | --- | --- |
| 1 | 장표 생성 | `준비` | 응답 준비/대기 안내 |
| 2 | `응답 열기` | `응답 중` | 입력 폼 표시 |
| 3 | `응답 마감` | `응답 마감` | 추가 제출 차단 |
| 4 | `결과 공개` | `결과 공개` | 공개 가능한 집계 표시 |
| 5 | `결과 숨기기` | `응답 마감` | 집계 숨김 |
| 6 | `응답 다시 열기` | `응답 중` | 기존 응답을 유지한 채 수정 가능 |

## 3. 사전 질문

### 3.1 예시 입력값

| 항목 | 입력값 |
| --- | --- |
| 제목 | `발표 전 질문` |
| 설명 | `발표 전에 궁금한 점을 남겨주세요.` |
| 문항 1 | `리허설 음성 분석은 어떻게 동작하나요?` |
| 문항 1 필수 여부 | 필수 |
| 문항 2 | `Orbit에서 가장 먼저 보고 싶은 기능은 무엇인가요?` |
| 문항 2 필수 여부 | 선택 |
| 응답자 이름 입력 허용 | 켬 |

청중 입력 예시:

| 청중 | 이름 | 문항 1 답변 | 문항 2 답변 |
| --- | --- | --- | --- |
| A | `테스트 청중 A` | `습관어도 분석 결과에 포함되나요?` | `실시간 발표 피드백` |
| B | `테스트 청중 B` | `리허설 결과는 어디에서 확인하나요?` | 입력하지 않음 |

### 3.2 에디터 구성 순서

1. `+` → `사전 질문`을 선택한다.
2. 제목과 설명을 예시 값으로 바꾼다.
3. 기본 문항 1을 예시 질문으로 바꾸고 `필수 문항`을 켠다.
4. `문항 추가 (1/5)`를 누른다.
5. 문항 2를 예시 질문으로 바꾸고 `필수 문항`은 끈다.
6. `응답자 이름 입력 허용`을 켠다.
7. 문항 2의 위·아래 이동 버튼을 눌러 순서 변경과 캔버스 반영을 확인한 뒤 원래 순서로 되돌린다.

완료 체크:

- [ ] 기본 주관식 문항 1개가 생성된다.
- [ ] 문항은 1개에서 5개까지 추가할 수 있다.
- [ ] 문항이 1개일 때 삭제 버튼이 비활성화된다.
- [ ] 문항 순서를 바꾸면 캔버스 카드 순서도 바뀐다.
- [ ] 제목, 설명, 두 문항이 캔버스와 썸네일에 표시된다.
- [ ] 새로고침 후 입력값과 순서가 유지된다.

### 3.3 청중 응답 순서

1. 발표자 화면에서 `응답 열기`를 누른다.
2. `장표별 직접 링크 복사` 또는 `QR 코드 확인`으로 청중 A 화면을 연다.
3. 이름과 문항 1을 비운 채 제출해 필수 검사를 확인한다.
4. 청중 A 예시 답변을 입력하고 제출한다.
5. 다른 브라우저에서 같은 링크를 열어 청중 B 예시 답변을 제출한다.
6. 청중 A 화면을 새로고침하고 기존 제출 내용이 유지되는지 확인한다.
7. 응답이 열린 상태에서 청중 A의 문항 1 답변을 `습관어와 침묵 구간도 포함되나요?`로 바꾸고 다시 제출한다.

완료 체크:

- [ ] 에디터와 같은 질문이 같은 순서로 표시된다.
- [ ] 필수 문항을 비우면 해당 문항 아래 오류가 표시되고 제출되지 않는다.
- [ ] 선택 문항은 비워도 제출된다.
- [ ] 이름을 비우면 익명으로 제출할 수 있다.
- [ ] 제출 후 접수 완료 화면이 표시된다.
- [ ] 새로고침 후 자신의 기존 응답이 복원된다.
- [ ] 같은 청중이 답변을 수정해도 응답 수는 중복 증가하지 않는다.

### 3.4 발표자 검수와 결과 공개 순서

1. 에디터의 응답 수가 `2`인지 확인한다.
2. 주관식 응답 목록에서 청중 A 응답을 `승인`한다.
3. 청중 B 응답을 `숨김`으로 바꾼다.
4. 청중 A 응답을 `답변 완료`로 바꾼다.
5. `응답 마감`을 누르고 청중 A가 더 이상 수정할 수 없는지 확인한다.
6. `결과 공개`를 누른다.
7. 청중 화면과 연결 결과 장표에서 승인된 청중 A 응답만 보이는지 확인한다.
8. `결과 숨기기`를 누르고 청중 결과가 다시 숨겨지는지 확인한다.
9. 첫 응답 이후 비활성화된 문항 설정과 `새 실행 버전 만들기` 안내를 확인한다.
10. `새 실행 버전 만들기`를 실행하고 새 버전의 응답 수가 `0`인지 확인한다.

완료 체크:

- [ ] 응답 수와 실제 고유 청중 수가 같다.
- [ ] `승인`, `숨김`, `답변 완료` 상태가 저장된다.
- [ ] 숨긴 답변은 청중 공개 결과와 연결 결과 장표에 나타나지 않는다.
- [ ] 응답 마감 중에는 새 제출과 수정이 차단된다.
- [ ] 새 실행 버전은 이전 응답을 보존하고 집계를 0부터 시작한다.

### 3.5 현재 별도 범위

- `[BLOCKED]` 발표자가 외부에서 미리 받은 질문을 파일 또는 패널에서 일괄 등록하는 기능

현재 구현은 청중 링크를 통한 제출과 한 건씩의 검수 흐름을 지원한다. 사전 수집 목록 일괄 업로드는 별도 기능으로 구현해야 한다.

## 4. 실시간 투표

### 4.1 예시 입력값

| 항목 | 입력값 |
| --- | --- |
| 제목 | `실시간 투표` |
| 설명 | `가장 먼저 사용해 보고 싶은 기능을 골라주세요.` |
| 질문 | `발표 준비에서 가장 도움이 될 기능은 무엇인가요?` |
| 문항 설정 | 필수, 단일 선택 |
| 선택지 1 | `AI 대본 생성` |
| 선택지 2 | `슬라이드 리허설` |
| 선택지 3 | `실시간 발표 보조` |
| 선택지 4 | `발표 리포트` |

권장 제출값:

- 청중 A: `슬라이드 리허설`
- 청중 B: `실시간 발표 보조`
- 청중 C: `슬라이드 리허설`

### 4.2 에디터 구성 순서

1. `+` → `실시간 투표`를 선택한다.
2. 제목, 설명, 질문을 예시 값으로 바꾼다.
3. 기본 선택지 2개의 문구를 수정한다.
4. `선택지 추가`를 두 번 눌러 선택지 3과 4를 입력한다.
5. 선택지 오른쪽의 위·아래 이동 버튼으로 순서를 바꾸고 캔버스 순서도 같이 바뀌는지 확인한다.
6. 선택지를 8개까지 추가했을 때 추가 버튼이 비활성화되는지 확인한다.
7. 다시 4개가 되도록 추가 선택지를 삭제한다.

완료 체크:

- [ ] 단일 선택 문항 1개만 존재하고 문항 추가·삭제 UI는 보이지 않는다.
- [ ] 선택지는 최소 2개가 유지되고 최대 8개까지 추가할 수 있다.
- [ ] 선택지 순서를 바꾸면 캔버스와 청중 화면에도 같은 순서로 표시된다.
- [ ] 선택지를 편집·추가·삭제하면 캔버스에 즉시 반영된다.
- [ ] 제목, 설명, 질문, 선택지가 새로고침 후 유지된다.

### 4.3 청중 응답 순서

1. `응답 열기`를 누른다.
2. 서로 다른 청중 브라우저 A, B, C에서 링크 또는 QR로 입장한다.
3. 아무 선택 없이 제출해 필수 검사를 확인한다.
4. 각 청중이 권장 제출값을 선택하고 제출한다.
5. 청중 B가 응답이 열린 동안 선택을 `발표 리포트`로 바꾸어 다시 제출한다.

완료 체크:

- [ ] 선택지 하나만 선택할 수 있다.
- [ ] 선택하지 않으면 제출되지 않고 오류가 표시된다.
- [ ] 제출 후 자신의 선택이 접수 완료 화면에 표시된다.
- [ ] 응답 수정 후 응답 수는 3으로 유지되고 집계만 바뀐다.

### 4.4 발표자 집계와 결과 공개 순서

1. 에디터 응답 수가 `3`인지 확인한다.
2. 결과 공개 전 청중 화면에 선택지별 집계가 노출되지 않는지 확인한다.
3. `응답 마감` 후 청중 C가 선택을 바꿀 수 없는지 확인한다.
4. `결과 공개`를 누른다.
5. 청중 결과와 연결 결과 장표에서 아래 최종 집계를 확인한다.
   - `슬라이드 리허설`: 2표, 66.7%
   - `발표 리포트`: 1표, 33.3%
   - 나머지 선택지: 0표
6. `결과 숨기기` 후 청중 집계가 숨겨지는지 확인한다.
7. `응답 다시 열기` 후 기존 3개 응답이 유지되는지 확인한다.

완료 체크:

- [ ] 선택지별 응답 수의 합이 전체 응답 수와 같다.
- [ ] 비율의 합이 반올림 오차 범위에서 100%다.
- [ ] 결과 공개 전에는 집계가 청중에게 노출되지 않는다.
- [ ] 결과 공개 후 청중 화면과 연결 결과 장표의 값이 같다.
- [ ] 새 실행 버전의 집계가 이전 버전과 섞이지 않는다.

## 5. 만족도 조사

### 5.1 예시 입력값

| 번호 | 유형 | 필수 | 질문 | 추가 설정 |
| --- | --- | --- | --- | --- |
| 1 | 5점 척도 | 예 | `오늘 발표는 전반적으로 만족스러웠나요?` | 왼쪽 `전혀 아니요`, 오른쪽 `매우 만족` |
| 2 | 단일 선택 | 예 | `가장 도움이 된 내용은 무엇인가요?` | `AI 대본 생성`, `슬라이드 리허설`, `발표 리포트` |
| 3 | 복수 선택 | 아니요 | `추가로 보고 싶은 자료를 모두 골라주세요.` | `기능 가이드`, `도입 사례`, `가격 안내` |
| 4 | 주관식 | 아니요 | `발표에서 개선되었으면 하는 점을 알려주세요.` | 없음 |

장표 내용:

- 제목: `발표 만족도 조사`
- 설명: `발표가 끝난 뒤 1분만 참여해 주세요.`

청중 입력 예시:

| 청중 | 5점 척도 | 가장 도움이 된 내용 | 추가로 보고 싶은 자료 | 개선 의견 |
| --- | --- | --- | --- | --- |
| A | `4` | `슬라이드 리허설` | `기능 가이드`, `도입 사례` | `실제 리허설 결과 예시를 더 보고 싶어요.` |
| B | `5` | `발표 리포트` | `기능 가이드` | `리포트 지표 설명이 더 자세하면 좋겠습니다.` |
| C | `3` | `슬라이드 리허설` | `도입 사례`, `가격 안내` | `청중 참여 방법을 먼저 안내해 주세요.` |

위 입력값을 그대로 제출했을 때의 예상 집계:

- 총 응답: `3`
- 5점 척도 평균: `4.0`
- 5점 척도 분포: `1점 0명`, `2점 0명`, `3점 1명`, `4점 1명`, `5점 1명`
- 단일 선택: `슬라이드 리허설 2명(66.7%)`, `발표 리포트 1명(33.3%)`, `AI 대본 생성 0명`
- 복수 선택: `기능 가이드 2명`, `도입 사례 2명`, `가격 안내 1명`
- 주관식 원문: `3개`이며 승인된 응답만 공개 결과에 표시

### 5.2 에디터 구성 순서

1. `+` → `만족도 조사`를 선택한다.
2. 제목과 설명을 예시 값으로 바꾼다.
3. 기본 5점 척도 문항의 질문과 양쪽 label을 수정한다.
4. 기본 주관식 문항을 `단일 선택`으로 바꾸고 선택지 3개를 만든다.
5. `문항 추가`를 누른 뒤 새 문항을 `복수 선택`으로 바꾸고 선택지 3개를 만든다.
6. 복수 선택 문항의 `최대 선택 수`를 `2개`로 설정한다.
7. 단일·복수 선택지의 위·아래 이동 버튼으로 선택지 순서를 바꾼다.
8. `문항 추가`를 다시 눌러 주관식 문항을 만든다.
9. 각 문항의 필수 여부를 예시와 맞춘다.
10. 위·아래 이동으로 문항 순서를 바꾸고 캔버스 순서도 바뀌는지 확인한 뒤 원래 순서로 되돌린다.

완료 체크:

- [ ] 기본 문항으로 5점 척도와 주관식이 생성된다.
- [ ] 문항은 최소 1개, 최대 5개까지 설정할 수 있다.
- [ ] 문항 유형을 `5점 척도`, `단일 선택`, `복수 선택`, `주관식`으로 바꿀 수 있다.
- [ ] 유형에 맞는 label 또는 선택지 설정만 표시된다.
- [ ] 복수 선택 문항의 최대 선택 수를 1개부터 전체 선택지 수까지 설정할 수 있다.
- [ ] 단일·복수 선택지 순서를 바꾸면 캔버스와 청중 화면에 같은 순서로 표시된다.
- [ ] 문항 순서와 전체 질문이 캔버스에 같은 순서로 표시된다.

### 5.3 청중 응답 순서

1. `응답 열기`를 누른다.
2. 링크 또는 QR로 청중 A 화면을 연다.
3. 필수 5점 척도와 단일 선택을 비우고 제출해 오류를 확인한다.
4. 복수 선택 문항에서 3개를 선택해 `최대 2개` 제한이 동작하는지 확인한다.
5. 청중 A 예시 응답을 입력하고 제출한다.
6. 새로고침 후 기존 응답이 복원되는지 확인한다.
7. 응답이 열린 동안 5점 척도를 `5`로 바꾸어 저장한 뒤 다시 `4`로 되돌려 제출한다.
8. 다른 브라우저에서 청중 B, C 예시 응답을 각각 제출한다.

완료 체크:

- [ ] 5점 척도는 1~5 중 하나만 선택할 수 있다.
- [ ] 단일 선택은 하나만 선택할 수 있다.
- [ ] 복수 선택은 여러 항목을 선택할 수 있다.
- [ ] 복수 선택은 설정한 최대 선택 수를 넘길 수 없다.
- [ ] 주관식에 일반 텍스트를 입력할 수 있다.
- [ ] 필수 문항을 비우면 문항별 오류가 표시된다.
- [ ] 선택 문항은 비워도 제출할 수 있다.
- [ ] 제출과 수정 후 접수 완료 화면에 최신 응답이 표시된다.

### 5.4 발표자 집계와 결과 공개 순서

1. 에디터 응답 수가 `3`인지 확인한다.
2. 5점 척도 평균 `4.0`과 1~5점 분포 `0, 0, 1, 1, 1`을 에디터 집계와 비교한다.
3. 단일 선택 집계가 `슬라이드 리허설 2`, `발표 리포트 1`, `AI 대본 생성 0`인지 확인한다.
4. 복수 선택 집계가 `기능 가이드 2`, `도입 사례 2`, `가격 안내 1`인지 확인한다.
5. 주관식 응답 중 하나를 `승인`, 하나를 `숨김`, 하나를 `답변 완료`로 바꾼다.
6. `응답 마감` 후 추가 제출이 차단되는지 확인한다.
7. `결과 공개` 후 청중 화면과 연결 결과 장표를 확인한다.
8. `결과 숨기기`와 `응답 다시 열기`를 순서대로 확인한다.

완료 체크:

- [ ] 5점 척도 평균과 분포가 원본 응답과 일치한다.
- [ ] 단일·복수 선택의 수와 비율이 원본 응답과 일치한다.
- [ ] 승인된 주관식만 공개 결과에 표시된다.
- [ ] 숨긴 응답은 공개 결과에 표시되지 않는다.
- [ ] 연결 결과 장표와 청중 결과의 값이 같다.
- [ ] 새 실행 버전은 이전 만족도 결과를 보존하고 0부터 시작한다.

## 6. 공통 회귀 검수

- [ ] 새로고침 후 장표 정의, 현재 실행 버전, 운영 상태가 유지된다.
- [ ] 장표를 이동해도 응답과 결과가 다른 참여 장표와 섞이지 않는다.
- [ ] 참여 장표 복제본은 원본과 다른 `activityId`를 사용한다.
- [ ] 결과 장표는 선택한 원본 장표의 최신 실행 결과만 표시한다.
- [ ] 원본 참여 장표 삭제 시 연결 결과 장표가 복구 가능한 오류 상태를 표시한다.
- [ ] 직접 링크와 QR은 같은 `sessionId`, `activityId`로 연결된다.
- [ ] 결과 비공개 상태에서 청중 API와 화면에 집계가 노출되지 않는다.
- [ ] 청중 화면에 발표자 대본, raw audio, 비공개 주관식 응답이 노출되지 않는다.
- [ ] 390×844 모바일 화면에서 문항, 선택지, 제출 버튼이 잘리거나 겹치지 않는다.
- [ ] 1024×768과 1440×900에서 에디터 설정·운영 패널이 가로로 잘리지 않는다.
- [ ] 키보드만으로 문항 선택, 입력, 제출과 운영 버튼 실행이 가능하다.
- [ ] 네트워크 오류 시 오류 문구와 재시도 가능한 동작이 표시된다.
