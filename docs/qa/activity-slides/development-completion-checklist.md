# 참여 장표 개발 완료 검수 체크리스트

## 1. 목적

이 문서는 에디터에서 `사전 질문`, `실시간 투표`, `만족도 조사` 장표를 만든 뒤 청중 응답과 결과 공개까지 정상 동작하는지 수동으로 확인하기 위한 체크리스트다.

각 시나리오는 아래 범위를 공통으로 검증한다.

- 에디터 입력값이 캔버스와 썸네일에 반영되는가
- 발표 세션을 만들고 장표별 응답을 열 수 있는가
- 직접 링크와 QR로 청중 화면에 접근할 수 있는가
- 청중이 응답을 제출하고, 응답이 에디터에 반영되는가
- 응답 마감, 재오픈, 결과 공개와 숨김이 동작하는가
- 첫 응답 이후 문항 정의가 잠기고 새 실행 버전을 만들 수 있는가
- 결과 연결 장표가 현재 실행 결과를 올바르게 보여주는가

## 2. 사전 준비

- Web, API, DB가 모두 실행 중이어야 한다.
- 발표자용 에디터와 청중용 브라우저를 분리한다.
  - 발표자: 로그인된 일반 브라우저
  - 청중: 시크릿 창 또는 다른 기기
- 서로 다른 청중으로 집계하려면 각각 다른 브라우저 프로필이나 기기를 사용한다.
- 에디터의 왼쪽 슬라이드 패널과 오른쪽 속성 패널을 열어 둔다.
- 이전 테스트 응답 때문에 문항 편집이 잠겨 있으면 `새 실행 버전`을 만들거나 새 참여 장표를 추가한다.

## 3. 공통 시작 순서

1. 프로젝트 에디터를 연다.
2. 왼쪽 슬라이드 패널 아래의 `+` 버튼을 누른다.
3. 검수할 참여 장표 유형을 선택한다.
   - `사전 질문`
   - `실시간 투표`
   - `만족도 조사`
4. 오른쪽 속성 패널에서 제목, 설명, 문항을 입력한다.
5. 입력한 내용이 캔버스와 왼쪽 썸네일에 즉시 반영되는지 확인한다.
6. 운영 영역에서 발표 세션이 없으면 `발표 세션 만들기`를 누른다.
7. 필요하면 왼쪽 `+` 메뉴에서 `참여 결과 연결`을 추가한다.
8. `응답 열기`를 눌러 청중 응답을 받기 시작한다.
9. `직접 링크 복사` 또는 `QR 코드 보기`로 청중 화면을 연다.

## 4. 사전 질문 검수

### 4.1 예시 입력

| 항목 | 예시 값 |
| --- | --- |
| 제목 | `발표 전 질문` |
| 설명 | `발표 전에 궁금한 점을 남겨주세요.` |
| 문항 1 | `발표 준비 과정에서 가장 궁금한 점은 무엇인가요?` |
| 문항 1 설정 | 필수 |
| 문항 2 | `Orbit에서 가장 먼저 보고 싶은 기능은 무엇인가요?` |
| 문항 2 설정 | 선택 |
| 응답자 이름 | 입력 허용 |

청중 테스트 응답:

- 문항 1: `리허설 음성 분석은 어떻게 동작하나요?`
- 문항 2: `실시간 발표 피드백 기능`
- 이름: `테스트 청중 A`

### 4.2 에디터 문항 설정

- [ ] `사전 질문` 장표를 추가하면 주관식 문항 1개가 기본으로 생성된다.
- [ ] `문항 추가`를 누르면 주관식 문항이 추가된다.
- [ ] 문항은 최소 1개, 최대 5개까지 설정할 수 있다.
- [ ] 문항이 1개일 때는 삭제할 수 없다.
- [ ] 위·아래 이동 버튼으로 문항 순서를 바꿀 수 있다.
- [ ] 필수 문항 설정을 켜고 끌 수 있다.
- [ ] 응답자 이름 입력 허용을 켜고 끌 수 있다.
- [ ] 제목, 설명, 모든 문항이 캔버스 질문 카드에 표시된다.
- [ ] 왼쪽 썸네일에서도 수정된 장표를 식별할 수 있다.

### 4.3 청중 응답

1. `응답 열기`를 누른다.
2. 직접 링크 또는 QR을 청중 브라우저에서 연다.
3. 필수 문항을 비우고 제출해 유효성 검사를 확인한다.
4. 위 예시 응답을 입력하고 제출한다.

- [ ] 청중 화면에 에디터에서 설정한 문항이 같은 순서로 보인다.
- [ ] 필수 문항을 비우면 제출할 수 없다.
- [ ] 선택 문항은 비워도 제출할 수 있다.
- [ ] 제출 후 접수 완료 상태가 표시된다.
- [ ] 응답이 열려 있는 동안 같은 청중은 응답을 수정할 수 있다.
- [ ] 같은 청중이 응답을 수정해도 응답 수가 중복 증가하지 않는다.

### 4.4 발표자 확인과 결과

- [ ] 제출 후 에디터 응답 수가 1 증가한다.
- [ ] 제출된 자유 입력 응답이 운영 또는 결과 영역에 표시된다.
- [ ] 응답을 `승인`, `숨김`, `답변 완료` 상태로 변경할 수 있다.
- [ ] 숨긴 응답은 공개 결과에 표시되지 않는다.
- [ ] `응답 마감` 후 청중은 새 응답을 제출하거나 수정할 수 없다.
- [ ] `응답 다시 열기` 후 기존 응답을 유지한 채 다시 제출할 수 있다.
- [ ] `결과 공개` 후 청중 화면과 결과 연결 장표에 승인된 응답만 표시된다.
- [ ] `결과 숨기기` 후 청중 화면에서 결과가 다시 숨겨진다.
- [ ] 첫 응답 이후 문항 편집이 잠긴다.
- [ ] `새 실행 버전`을 만들면 기존 결과를 보존하고 새 응답 수는 0에서 시작한다.

### 4.5 아직 별도 확인이 필요한 기능

- [ ] 발표자가 사전에 받은 질문 목록을 파일이나 패널에서 일괄 업로드할 수 있다.

현재 코드 기준으로 발표자가 기존 질문을 응답 데이터로 직접 업로드하는 전용 기능은 없다. 임시 검수는 발표자가 청중 링크를 열어 질문을 한 건씩 제출하는 방식으로만 가능하다.

## 5. 실시간 투표 검수

### 5.1 예시 입력

| 항목 | 예시 값 |
| --- | --- |
| 제목 | `실시간 투표` |
| 설명 | `현재 발표에서 가장 유용한 기능을 골라주세요.` |
| 질문 | `가장 먼저 사용해 보고 싶은 기능은 무엇인가요?` |
| 설정 | 필수, 단일 선택 |
| 선택지 1 | `AI 대본 생성` |
| 선택지 2 | `슬라이드 리허설` |
| 선택지 3 | `실시간 발표 보조` |
| 선택지 4 | `발표 리포트` |

### 5.2 에디터 문항 설정

- [ ] `실시간 투표` 장표를 추가하면 단일 선택 문항 1개가 생성된다.
- [ ] 투표 문항을 추가하거나 삭제하는 버튼이 노출되지 않는다.
- [ ] 선택지는 최소 2개, 최대 8개까지 설정할 수 있다.
- [ ] 빈 선택지나 중복 선택지가 저장되지 않는다.
- [ ] 제목, 설명, 질문, 선택지가 캔버스에 즉시 반영된다.
- [ ] 선택지 순서를 바꾸면 캔버스 순서도 함께 바뀐다.

### 5.3 청중 응답

1. `응답 열기`를 누른다.
2. 서로 다른 청중 브라우저 2개 이상에서 직접 링크 또는 QR을 연다.
3. 각 브라우저에서 다른 선택지를 제출한다.

권장 테스트 데이터:

- 청중 A: `슬라이드 리허설`
- 청중 B: `실시간 발표 보조`
- 청중 C: `슬라이드 리허설`

- [ ] 청중은 선택지 하나만 고를 수 있다.
- [ ] 필수 문항을 선택하지 않으면 제출할 수 없다.
- [ ] 제출 후 선택한 값이 접수 완료 화면에 유지된다.
- [ ] 응답이 열린 동안 선택을 바꾸고 다시 제출할 수 있다.
- [ ] 서로 다른 청중의 제출 건수가 각각 집계된다.

### 5.4 발표자 확인과 결과

- [ ] 제출할 때마다 에디터 응답 수가 갱신된다.
- [ ] 선택지별 응답 수와 비율이 정확하다.
- [ ] 결과 공개 전에는 청중에게 집계 결과가 보이지 않는다.
- [ ] `응답 마감` 후 추가 제출이 차단된다.
- [ ] `결과 공개` 후 청중 화면과 결과 연결 장표에 투표 결과가 표시된다.
- [ ] 위 권장 데이터 기준 `슬라이드 리허설` 2표, `실시간 발표 보조` 1표로 표시된다.
- [ ] `결과 숨기기` 후 공개 결과가 숨겨진다.
- [ ] 첫 응답 이후 질문과 선택지 편집이 잠긴다.
- [ ] `새 실행 버전`의 집계는 이전 실행 결과와 섞이지 않는다.

## 6. 만족도 조사 검수

### 6.1 예시 입력

| 번호 | 유형 | 필수 | 예시 문항 및 선택지 |
| --- | --- | --- | --- |
| 1 | 5점 척도 | 예 | `오늘 발표는 전반적으로 만족스러웠나요?` / 왼쪽 `전혀 아니요`, 오른쪽 `매우 만족` |
| 2 | 단일 선택 | 예 | `가장 도움이 된 내용은 무엇인가요?` / `AI 대본 생성`, `슬라이드 리허설`, `발표 리포트` |
| 3 | 다중 선택 | 아니요 | `추가로 보고 싶은 자료를 모두 골라주세요.` / `기능 가이드`, `도입 사례`, `가격 안내`, 최대 2개 |
| 4 | 주관식 | 아니요 | `발표에서 개선되었으면 하는 점을 알려주세요.` |

장표 제목과 설명:

- 제목: `발표 만족도 조사`
- 설명: `발표가 끝난 뒤 1분만 참여해 주세요.`

청중 테스트 응답:

- 만족도: `4점`
- 가장 도움이 된 내용: `슬라이드 리허설`
- 추가 자료: `기능 가이드`, `도입 사례`
- 개선 의견: `실제 리허설 결과 예시를 더 보고 싶어요.`

### 6.2 에디터 문항 설정

- [ ] `만족도 조사` 장표를 추가하면 기본 문항이 생성된다.
- [ ] 문항을 최소 1개, 최대 5개까지 추가할 수 있다.
- [ ] 문항을 삭제하고 위·아래로 순서를 바꿀 수 있다.
- [ ] 문항 유형을 `5점 척도`, `단일 선택`, `다중 선택`, `주관식`으로 변경할 수 있다.
- [ ] 유형 변경 시 해당 유형에 필요한 설정만 표시된다.
- [ ] 다중 선택의 최대 선택 수를 설정할 수 있다.
- [ ] 제목, 설명, 모든 문항이 캔버스에 같은 순서로 반영된다.

### 6.3 청중 응답

1. `응답 열기`를 누른다.
2. 직접 링크 또는 QR로 청중 화면을 연다.
3. 필수 문항을 비운 상태, 다중 선택 제한을 초과한 상태를 각각 검증한다.
4. 위 예시 응답을 입력하고 제출한다.

- [ ] 5점 척도는 1점부터 5점까지 하나만 선택할 수 있다.
- [ ] 단일 선택은 하나만 선택할 수 있다.
- [ ] 다중 선택은 설정한 최대 개수를 초과할 수 없다.
- [ ] 주관식 문항에 일반 텍스트를 입력할 수 있다.
- [ ] 필수 문항을 비우면 제출할 수 없다.
- [ ] 선택 문항은 비워도 제출할 수 있다.
- [ ] 제출 후 접수 완료 상태가 표시된다.

### 6.4 발표자 확인과 결과

- [ ] 응답 수가 정상 증가한다.
- [ ] 5점 척도의 평균과 분포가 정확하다.
- [ ] 단일·다중 선택의 선택지별 응답 수와 비율이 정확하다.
- [ ] 주관식 응답을 `승인`, `숨김`, `답변 완료`로 관리할 수 있다.
- [ ] 숨긴 주관식 응답은 공개 결과에서 제외된다.
- [ ] `응답 마감`, `응답 다시 열기`, `결과 공개`, `결과 숨기기`가 동작한다.
- [ ] 결과 연결 장표에 척도, 선택형 집계, 승인된 주관식 결과가 표시된다.
- [ ] 첫 응답 이후 문항 편집이 잠긴다.
- [ ] `새 실행 버전`을 만들면 새로운 만족도 집계가 0에서 시작한다.

## 7. 공통 회귀 검수

- [ ] 새로고침 후 장표 정의와 실행 상태가 유지된다.
- [ ] 장표를 이동해도 해당 장표의 응답과 결과가 다른 장표와 섞이지 않는다.
- [ ] 참여 장표를 복제했을 때 새로운 `activityId`를 사용한다.
- [ ] 참여 장표를 삭제한 뒤 연결된 결과 장표가 복구 가능한 오류 상태를 보여준다.
- [ ] 청중 링크는 발표자 대본이나 원본 음성 같은 비공개 정보를 노출하지 않는다.
- [ ] QR과 직접 링크가 같은 장표와 현재 실행 버전으로 연결된다.
- [ ] 결과 비공개 상태에서는 청중이 결과 API나 화면으로 집계를 볼 수 없다.
- [ ] 작은 모바일 화면에서 문항, 선택지, 제출 버튼이 잘리거나 겹치지 않는다.
- [ ] 키보드만으로 문항 이동, 선택, 제출이 가능하다.
- [ ] 오류 발생 시 재시도 방법이 화면에 명확하게 표시된다.

## 8. 개발 완료 판정표

| 기능 | 코드 기준 상태 | 완료 판정 조건 |
| --- | --- | --- |
| 사전 질문 1~5개 편집 | 구현 | 4.2 전체 통과 |
| 사전 질문 QR 실시간 수집 | 구현 | 4.3, 4.4 전체 통과 |
| 사전 수집 질문 일괄 업로드 | 미구현 | 파일 또는 패널 입력으로 응답 데이터 생성 가능 |
